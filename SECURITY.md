# Security Policy

## Supported Versions

Security fixes are accepted for the current `main` branch.

## Reporting A Vulnerability

Do not open public issues for secrets, private data exposure, or exploitable
security bugs. Contact the maintainer privately with:

- affected file or feature
- steps to reproduce
- expected impact
- suggested fix, if known

The maintainer will acknowledge valid reports, investigate, and publish a fix
or mitigation.

## Data Handling

Do not commit real patient records, credentials, tokens, API keys, or private
medical images. Use synthetic examples only.
