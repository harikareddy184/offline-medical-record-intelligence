# Constitution

## Principles

1. Offline-first: core medical parsing must work without network calls.
2. CPU-first: features must run on ordinary CPU hardware.
3. Privacy-first: examples must be synthetic and safe to commit.
4. Explainable output: JSON fields should be readable and documented.

## Quality Gates

Changes should keep tests, linting, type checks, security checks, and
documentation checks available through local commands and CI.
